Chain-of-Thought (CoT) Prompting

AI models generate results token-by-token. Asking the model to think step-by-step before arriving at a final answer prevents logical collapse and halts hallucinations.

Standard Challenge Question

"If a train travels 60 km/hr for 3 hours, how far does it travel?"

Prompt Modifier "Think step-by-step before answering."
💡 **Why it works**: Instructing the model to elaborate logic pathways establishes sequential memory states, giving it space to calculate parameters correctly.

Slide 4 / 8 (Slide 46)

Step-Back Prompting

Instead of tackling a narrow, complex problem raw, Step-Back Prompting forces the AI model to zoom out, identify the overarching rules/principles, and use them as guides.

This prevents the model from generating shortsighted symptoms-based content and promotes strategic systems thinking.

Structural Workflow Sequence
1. Broad Principles 2. Specific Solution
Interactive Strategy Lens

Solving Symptom Directly

Focuses strictly on surface parameters

[❌ Direct Prompt]

"How do we reduce plastic pollution?"

Yields generic surface ideas: "Use paper straws, pick up litter." Targets localized symptoms rather than core system causes.

Slide 5 / 8 (Slide 47)

Prompt Chaining

Complex procedures suffer when forced into a single execution prompt. Prompt Chaining resolves this by splitting the scope into sequential links. The output of Prompt A directly feeds the parameters of Prompt B:

Maintains highly optimal token window spaces
Permits strict QA verification boundaries
Dramatically decreases cumulative reasoning drift
