Background & History

What is Pandas?

Pandas is an open-source Python library providing highly optimized data structures and operational utilities designed for seamless tabular analysis.

Etymology

The name is derived from the econometrics concept Panel Data (multidimensional tracking arrays), developed originally in 2008 by Wes McKinney for commercial quantitative finance environments.

The Five Data Lifecycles

1. LOAD
2. PREPARE
3. MANIPULATE
4. MODEL
5. ANALYZE

Why was Pandas needed?

  • Prior to Pandas: Python was primarily used for data munging and preparation, but was inefficient for deep statistical exploration.
  • The Solution: Introducing fast, labeled multidimensional structures natively bound with analytical computation.
  • Versatility: Broad academic & industrial adoption ranging across finance, biomedical stats, physics modeling, and business intelligence systems.

Data Structure I

Pandas Series

A Series is a 1D labeled array capable of holding data of any type (integers, strings, floats, Python objects). Think of it as a single column in a worksheet.

Creating a Series with Explicit String Indices: 
import pandas as pd
a = [1, 7, 2]
my_series = pd.Series(a, index=["x", "y", "z"])
print(my_series)
From Python Dictionary (Keys become Index): 
calories = {"day1": 420, "day2": 380}
my_series = pd.Series(calories)
print(my_series)

Data Structure II

Pandas DataFrame

A DataFrame is a 2D labeled spreadsheet structure with mutable size, row labels (index), and column headers that support multiple divergent datatypes.

It can be constructed from nested dictionaries, lists of dictionaries, lists of tuples, multidimensional NumPy arrays, or loaded from disk.

Standard constructor structure: 
df = pd.DataFrame(
    data,
    index=row_labels,
    columns=col_headers,
    dtype=type_override
)
Selecting Rows & Columns:
  • df["col"]: Selects single column as Series.
  • df.loc["row_label"]: Accesses row by string label index.
  • df.iloc[0]: Accesses row by positional integer index.

Methodology Explanation

Wrong Format & Wrong Data Deep-Dive

Problem 1: Formatting

Converting Chronological Formats

Raw inputs often represent dates as unparsed strings or inconsistent formats (e.g. 20201226 vs '2020/12/26'). Pandas easily resolves this with pd.to_datetime().

PYTHON SOLUTION 
# Convert strings directly to datetime64
df['Date'] = pd.to_datetime(df['Date'])

Note: Empty date targets default to NaT (Not a Time) structures, which can subsequently be targeted for removal using dropna().

Problem 2: Outliers

Fixing Anomaly Values (Outliers)

Wrong values (such as an accidental extra zero making a workout session 450 minutes instead of 45) skew aggregations and summaries. We resolve this using index location mapping or conditional ceiling thresholds.

METHOD A: DIRECT CELL MAPPING 
df.loc[7, 'Duration'] = 45
METHOD B: CONDITIONAL REPLACEMENT LIMITER 
for x in df.index:
    if df.loc[x, "Duration"] > 120:
        df.loc[x, "Duration"] = 120
